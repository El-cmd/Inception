Projet d'infrastructure avec Docker

Ce projet consiste à mettre en place une petite infrastructure composée de différents services selon des règles spécifiques.

Tâches :

  • Mettre en place Docker Compose
  • Créer des Dockerfiles pour chaque service
  • Construire les images Docker
  • Configurer un conteneur NGINX avec TLSv1.2 ou TLSv1.3
  • Configurer un conteneur WordPress + php-fpm sans nginx
  • Configurer un conteneur MariaDB sans nginx
  • Créer des volumes pour la base de données WordPress et les fichiers du site
  • Établir un réseau Docker entre les conteneurs
  • Faire redémarrer les conteneurs en cas de plantage
  • Configurer deux utilisateurs dans la base de données WordPress
  • Configurer les volumes pour être accessibles sur la machine hôte
  • Configurer le nom de domaine pour pointer vers l'adresse IP locale
  • Configurer le conteneur NGINX comme seul point d'entrée avec le port 443 et TLSv1.2 ou TLSv1.3
